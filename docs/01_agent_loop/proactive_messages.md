# 主动消息

## 设计目标

- 定义 Agent 在任务执行过程中主动反馈的场景与约束。
- 保证主动消息受权限与预算治理。

## 核心概念

- **主动消息**：在无用户新输入时，由 Agent 发送的状态更新或建议。
- **触发条件**：任务完成、阻塞、预算不足、冲突提示等。
- **消息类型**：状态同步、请求确认、冲突曝光、建议下一步。

## 设计约束

- 主动消息必须经过权限校验与预算配额。
- 不允许自动覆盖冲突；必须向用户说明并请求处理。
- 主动消息内容应引用事实与审计信息来源。

## 非目标

- 不定义具体消息渠道格式或 UI 交互细节。

## 未来演进方向

- 支持可配置的消息节流与优先级策略。
- 与 Web Console 的任务通知机制联动。
