# Fact 生命周期

## 设计目标

- 明确事实从提议到确认的全流程。
- 保证事实可审计、可撤销、可过期。

## 生命周期状态

```
Proposal → (Accept | Reject | Expire | Revoke)
```

- **Proposal**：事实被提出，等待验证或确认。
- **Accept**：事实被采纳，进入可注入的记忆集合。
- **Reject**：事实被否决，不参与记忆。
- **Expire**：事实过期失效，但保留审计记录。
- **Revoke**：事实被撤销，需说明撤销原因与来源。

## 状态转移原则

- 任何状态变更必须记录来源与时间戳。
- Fact 不能被静默覆盖，只能通过显式状态转移。
- Expire 与 Revoke 必须暴露给用户或管理员。

## 非目标

- 不定义事实验证的算法或策略细节。

## 未来演进方向

- 引入多方确认与可信度评分机制。
