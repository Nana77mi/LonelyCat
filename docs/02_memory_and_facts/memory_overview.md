# 记忆与事实概览

## 设计目标

- 将 Facts/Memory 作为一等公民纳入 Agent Loop。
- 确保长期记忆具备可审计与可控生命周期。

## 核心概念

- **Fact**：具备来源、范围、生命周期与审计记录的事实。
- **Memory**：由 Facts 聚合形成的可查询上下文集合。
- **Fact Store**：事实与审计记录的持久化载体。

## 设计约束

- Facts 必须注入 system message，而非 user message。
- 记忆读取必须遵循 scope 与权限规则。
- 任何冲突必须显式暴露给用户。

## 非目标

- 不定义具体存储结构或向量检索实现。

## 未来演进方向

- 支持跨任务的记忆合并策略与冲突评分。
