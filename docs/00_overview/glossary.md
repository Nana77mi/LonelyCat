# 术语表

## 设计目标

- 统一概念边界，减少实现偏差。
- 为跨团队协作提供一致语言。

## 术语

- **Agent Loop**：决策 → 执行 → 主动消息的循环过程。
- **Planner**：负责将输入转化为可执行计划的组件角色。
- **Executor**：负责执行计划并调用工具的组件角色。
- **Skills**：LonelyCat 内部统一工具抽象。
- **MCP**：外部工具接入协议，视作外部 Skill Provider。
- **Fact**：可审计的事实记录，具有生命周期与范围。
- **Memory**：基于 Facts 形成的长期上下文集合。
- **Scope**：事实与记忆的作用域，包含 global/session/project。
- **Connector**：消息与事件的接入层，如 QQ/WeChat/Web。
- **Web Console**：管理任务、工具、记忆与审计的控制台。

## 非目标

- 不尝试定义每个词的实现细节与字段级结构。

## 未来演进方向

- 随系统演进补充更多术语，如多 Agent 协作与跨域权限。
