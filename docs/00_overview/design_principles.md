# 设计原则

## 设计目标

- 保证 Agent 平台长期演进时的稳定性与可控性。
- 降低多端接入与多工具扩展带来的复杂度。
- 以事实与记忆为核心，确保行为可解释、可审计。

## 核心原则

1. **统一抽象优先**
   - Skills 是唯一工具抽象，MCP 只是外部提供者。
   - Agent/Planner/Executor 对工具来源无感。
2. **预算与权限先行**
   - 工具调用、消息发送与记忆读写必须在预算与权限框架内。
3. **可观测与可审计**
   - 所有关键行为必须可回放、可追责、可标注。
4. **事实优先、记忆一等公民**
   - Facts 有明确生命周期与冲突处理，不允许静默覆盖。
5. **本地优先**
   - 以本地运行与控制为基线，再扩展远程能力。

## 非目标

- 不保证单一实现语言或框架。
- 不为每个 Connector 提供完全一致的体验。

## 未来演进方向

- 形成可插拔的治理策略库（预算、审计、权限）。
- 逐步提升多 Agent 协作能力并保持可控。
