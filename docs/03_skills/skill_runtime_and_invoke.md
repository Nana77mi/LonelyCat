# Skill 运行时与调用

## 设计目标

- 明确 Skill 的运行时职责与调用路径。
- 保证一致的调用治理与结果表达。

## 核心概念

- **Runtime**：加载、路由并执行 Skill 的运行时层。
- **Invocation**：对 Skill 的一次调用，包含上下文与审计信息。

## 调用流程（文字描述）

1. Executor 发起调用请求并携带计划上下文。
2. Runtime 校验权限、预算与参数 Schema。
3. 执行 Skill 并生成结构化结果或错误。
4. 结果与审计记录写入日志与任务状态。

## 设计约束

- Invocation 必须包含请求 ID、调用者与上下文引用。
- 结果必须可追踪到 Manifest 版本。

## 非目标

- 不定义并发模型或缓存策略细节。

## 未来演进方向

- 引入“可复现调用”以支持回放与调试。
