# Connectors

## 设计目标

- 支持多端接入与统一消息治理。
- 统一身份映射与事件处理。

## 核心概念

- **Connector**：与 QQ/WeChat/Web 等渠道的对接层。
- **Identity Mapping**：将渠道身份映射为平台用户身份。
- **Event Bridge**：将外部事件转换为平台事件。

## 设计约束

- Connector 不直接调用工具，只负责消息收发。
- 所有消息必须经过 Core API 进行权限与预算校验。

## 非目标

- 不定义具体协议实现（如 OneBot 版本细节）。

## 未来演进方向

- 支持统一事件规范与多端一致性策略。
